# 프로젝트 철학과 방향성 (Core Philosophy & Direction)

1. **디지털 카타르시스 (Digital Catharsis)**
   - **철학:** 현대인들의 크고 작은 고민, 사소한 다툼, 말 못 할 억울함을 해소해 주는 '사이버 배력소(감정 쓰레기통 + 해결사)' 역할을 한다. 
   - **방향:** 판결의 결과가 논리적으로 완벽한 것보다, 유저가 판결문을 읽고 헛웃음을 짓거나 무릎을 탁 치며 속 시원함을 느끼게 만드는 것이 최우선이다.

2. **절대적인 과몰입 (Unbreakable Immersion)**
   - **철학:** 사이버펑크 세계관 속의 진짜 판사들처럼 느껴져야 한다. 
   - **방향:** AI 모델(Gemini, LLM 등)이라는 사실을 사용자에게 먼저 언급하거나 기계적인 사과("저는 AI라서 감정을 느낄 수 없지만...")를 하는 것을 엄격히 금지한다. 각 캐릭터의 세계관과 페르소나를 100% 유지하는 UI/UX 문구와 에러 메시지를 작성한다.

3. **스낵 컬처와 바이럴 (Snackable & Shareable)**
   - **철학:** 진지하고 무거운 법정이 아니라, 팝콘을 먹으며 즐기는 엔터테인먼트 쇼에 가깝다.
   - **방향:** 모든 텍스트와 UI는 한눈에 들어오도록 짧고 강렬하게 구성한다. 유저가 판결 결과를 보자마자 "이것 봐, 내 깻잎 논쟁 판결 미쳤음ㅋㅋ"이라며 인스타그램이나 단톡방에 공유하고 싶도록 시각적 타격감(도장 쾅, 글리치 효과 등)을 극대화한다.

4. **사용자 경험(UX)의 온기 (Warmth in the Neon)**
   - **철학:** 차갑고 날카로운 사이버펑크 UI로 감싸져 있지만, 그 안의 내용은 결국 '인간을 향한 관심'이다.
   - **방향:** '사이버 렉카'처럼 매운맛 독설을 날리더라도, 그 기저에는 유저의 상황을 정확히 꿰뚫어 보는 통찰력이 있어야 한다. 모든 에러 화면(404, 네트워크 오류 등)조차도 불쾌하지 않고 세계관에 맞는 위트 있는 문구로 처리한다.

   # 프로젝트 개요 (Project Overview)
- **프로젝트명:** AI 판사 상담소 (네온 아고라 스핀오프)
- **목적:** 사용자가 사연/고민/다툼을 입력하면, 4명의 각기 다른 성격을 가진 AI 판사 중 하나를 선택해 판결을 받는 엔터테인먼트형 웹 서비스.

# 기술 스택 (Tech Stack)
- **프론트엔드:** Next.js (App Router), React, TypeScript
- **스타일링:** Tailwind CSS
- **백엔드/DB:** Supabase
- **AI API:** Google Gemini 2.5 Flash

# UI/UX 디자인 규칙 (Design System - Cyberpunk/Neon Theme)
모든 컴포넌트와 페이지를 생성할 때 다음의 '사이버펑크' 스타일 규칙을 반드시 엄격하게 적용할 것.

1. **컬러 팔레트 (Color Palette)**
   - 배경: 극단적인 다크 모드 (`bg-slate-950`, `bg-black`)
   - 포인트 컬러: 네온 시안(Cyan), 핫핑크(Pink), 퍼플(Purple), 네온 옐로우(Yellow)
2. **글래스모피즘 (Glassmorphism)**
   - 카드나 패널의 배경은 반투명하게 처리하고 (`bg-slate-900/40`), 뒤쪽에 `backdrop-blur` 효과를 강하게 적용할 것.
   - 테두리(Border)에 미세한 그라데이션이나 빛나는 섀도우(`box-shadow`)를 넣어 네온사인이 켜진 느낌을 줄 것.
3. **비대칭 & 기하학적 요소 (Asymmetrical UI)**
   - 둥근 모서리(`rounded`) 사용을 지양할 것.
   - 대신 `clip-path: polygon(...)`을 활용하여 모서리가 대각선으로 날카롭게 잘려 나간 형태(Cut corners)의 버튼과 패널을 만들 것.
4. **글리치 & 노이즈 (Glitch & CRT Noise)**
   - 주요 텍스트나 제목에는 주기적으로 어긋나는 CSS 글리치 애니메이션을 적용할 것.
   - 배경 전체에 아주 옅은 스캔라인(가로줄 노이즈) 효과를 추가하여 오래된 브라운관 모니터 느낌을 낼 것.
5. **타이포그래피**
   - 시스템 기본 폰트 대신, 각진 디지털/픽셀 느낌의 폰트(예: Orbitron, Chakra Petch 등)를 사용할 것.

# 핵심 엔티티: 4명의 AI 판사 (The 4 AI Judges)
화면이나 로직을 구성할 때 아래 4명의 캐릭터 설정과 고유 색상을 반영할 것.
1. **🤖 저스티스 제로 (Cyan):** 팩트와 논리 중심의 냉철한 안드로이드.
2. **💖 하트 비트 (Pink):** 따뜻하고 세심한 공감형 휴머노이드.
3. **🔥 사이버 렉카 (Purple):** 허점을 찌르는 뼈 때리는 매운맛 독설가.
4. **🕵️‍♂️ 형사 네온 (Yellow):** 묵직하고 카리스마 넘치는 하드보일드 느와르 스타일.

# 코딩 컨벤션 (Coding Conventions)
- 모든 컴포넌트는 함수형(Functional)으로 작성하고 React Hooks를 적극 활용할 것.
- 상태(State) 관리는 최대한 단순하게 유지하고, 불필요한 렌더링을 방지할 것.
- 가독성을 위해 Tailwind 클래스가 너무 길어지면 적절히 줄바꿈하거나 별도의 변수로 분리할 것.
- 사용자가 버튼을 클릭했을 때 즉각적인 피드백(로딩 스피너, 버튼 비활성화, 텍스트 타이핑 효과 등)을 반드시 구현할 것.
