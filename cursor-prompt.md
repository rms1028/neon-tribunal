# 네온즈 공개재판소 UI 개선 요청

## 목표
현재 공개재판소 페이지를 **인스타그램 피드 스타일**로 전면 리디자인해줘.
기존 카드 리스트 + 캐러셀 구조를 버리고, 스크롤하면서 하나씩 사연과 판결을 탐색하는 몰입형 피드로 바꿔줘.

## 참고 파일
- 첨부한 `neon-court-feed.jsx`를 디자인 레퍼런스로 참고해. 이 파일의 UI 구조, 컬러, 레이아웃을 따라가되, 우리 프로젝트의 기술 스택과 컴포넌트 구조에 맞게 변환해줘.

---

## 구조 변경사항

### 1. 헤더
- 스크롤 시 축소되는 sticky 헤더
- 로고(NEON COURT) + 알림/설정 버튼
- 배경: `rgba(10,10,15,0.85)` + `backdrop-filter: blur(20px)`

### 2. 필터 탭
- 기존 2줄 chip → **가로 스크롤 1줄** 으로 변경
- sticky로 헤더 바로 아래 고정
- 탭: 전체 / 저스티스 제로 / 하트비트 / 사이버 벵카 / 형사 네온
- 활성 탭은 해당 판사 컬러로 하이라이트

### 3. 스토리 행 (새로 추가)
- 인스타 스토리처럼 상단에 AI 판사 4명의 원형 아이콘 + 명예의전당
- 가로 스크롤, 탭하면 해당 판사의 하이라이트/최근 판결로 이동
- 각 판사 컬러로 gradient ring 표시

### 4. 피드 카드 (핵심 변경)
기존 리스트 카드를 아래 구조로 교체:

```
┌─────────────────────────────┐
│ 👤 유저명  ⚖️판사태그  시간  ⋯ │  ← 헤더
│                             │
│ "사연 본문 텍스트..."        │  ← 본문 (80자 초과시 더보기)
│ #태그1 #태그2               │
│                             │
│ ┌─ VERDICT ──────────────┐ │  ← 판결 (접힘/펼침)
│ │ 유죄  판결 요약         │ │
│ │ (펼치면) 상세 판결 설명  │ │
│ │ 배심원 투표: 👍동의 👎반대│ │
│ └────────────────────────┘ │
│                             │
│ 🔥234 💀89 ❤️167 🤔45  💬78 🔖│  ← 리액션 바
└─────────────────────────────┘
```

- 더블탭 → 🔥 리액션 애니메이션
- 판결(VERDICT)은 기본 접힌 상태, 탭하면 펼침
- 배심원 투표(동의/반대)는 판결 상세 안에 배치
- 리액션 4종: 🔥(불) 💀(충격) ❤️(공감) 🤔(고민)

### 5. 하단 내비게이션 (새로 추가 또는 수정)
- 4탭: 홈 / 탐색 / 랭킹 / 프로필
- fixed bottom, blur 배경

### 6. FAB 버튼
- "⚡ 새 재판" 플로팅 버튼
- 우하단, 하단 내비 위에 위치
- 그라디언트: `linear-gradient(135deg, #00ffcc, #00cc99)`

## 디자인 토큰

### 컬러
```
배경: #0a0a0f
카드 배경: rgba(255,255,255,0.02)
구분선: rgba(255,255,255,0.06)

판사 컬러:
- 저스티스 제로: #00ffcc
- 하트 비트: #ff2d78  
- 사이버 벵카: #7b5cff
- 형사 네온: #ffe14d

판결 상태:
- 유죄: #ff2d78 / bg rgba(255,45,120,0.15)
- 무죄: #00ffcc / bg rgba(0,255,204,0.15)
- 판결불가: #ffe14d / bg rgba(255,225,77,0.15)
```

### 타이포그래피
- 유저명: 14px / bold
- 사연 본문: 15px / line-height 1.65
- 태그: 12px / #7b5cff
- 판결 요약: 13px / bold
- 리액션 수: 12px

### 간격
- 카드 패딩: 좌우 16px
- 카드 간 간격: border-bottom으로 구분 (gap 아님)
- 하단 패딩: 최소 140px (FAB + 하단 내비 가림 방지)

## 주의사항
- 기존 API 연동 로직은 유지하고 UI 레이어만 교체해줘
- 기존 데이터 모델(사연, 판결, 투표 등)에 맞게 매핑해줘
- 모바일 우선, max-width 430px 기준
- 다크 테마 고정
- 기존 컴포넌트 분리 구조가 있으면 그대로 따르되, 없으면 적절히 분리해줘
