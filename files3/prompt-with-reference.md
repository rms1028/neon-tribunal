# 공개 재판소 UI 개선 프롬프트 (참조 코드 활용 버전)

`[대괄호]` 부분을 본인 상황에 맞게 수정한 뒤, **참조 UI 코드**와 **기존 코드**를 함께 첨부해서 보내세요.

---

## 프롬프트

```
나는 "NEONS"이라는 AI 공개 재판소 웹서비스를 만들고 있어.
기술 스택은 [Next.js + TypeScript + Tailwind / React + styled-components / 기타 본인 스택] 이야.

공개 재판소 페이지의 UI를 개선하려고 하는데,
내가 원하는 방향으로 만든 **참조용 UI 코드**를 첨부할게.

## 첨부 파일
1. `reference-ui.jsx` — 내가 원하는 UI/UX 방향을 보여주는 참조 코드 (React 데모)
2. `[현재 파일명]` — 실제 프로젝트의 기존 코드

## 요청 사항
참조 코드의 **디자인 패턴과 UX 흐름**을 기존 코드에 적용해줘.
참조 코드를 그대로 복붙하는 게 아니라, 아래 핵심 요소들을 기존 코드 구조에 맞게 녹여줘.

### 참조 코드에서 가져올 핵심 요소

1. **리스트에서 바로 투표 가능한 구조**
   - 카드 안에 판결 요약 + 투표 바 + 투표 버튼 바로 노출
   - 페이지 이동 없이 즉시 투표 가능

2. **댓글 인라인 토글**
   - 💬 아이콘 클릭 시 카드 내부에서 댓글이 펼쳐짐
   - 2개 미리보기 + "N개 더 보기" 확장
   - 입력창이 상단에 위치

3. **투표 피드백 애니메이션**
   - 버튼 텍스트 "동의" → "동의함 ✓" 변경
   - 투표 바 성장 애니메이션 + 글로우 효과

4. **댓글 좋아요 기능**
   - 각 댓글에 하트 버튼 + 카운트

5. **싱글 컬럼 레이아웃**
   - 기존 2컬럼(투표/댓글 분리) → 싱글 컬럼 흐름
   - 사연 → 판결 → 투표 → 댓글 순서

6. **시각적 요소**
   - 판사별 컬러 시스템 (초록/파랑/핑크/보라)
   - HOT 뱃지, 탭 필터, staggered 진입 애니메이션
   - 다크 테마 + 네온 컬러 액센트

### 유지해야 할 것
- 기존 코드의 API 호출 로직
- 기존 라우팅 구조
- 기존 상태관리 방식 [Redux / Zustand / Context 등]
- 기존 인증/권한 처리

### 디자인 토큰 (참조 코드 기준)
- 배경: #0a0a0f
- 메인 액센트: #39FF14 (네온 그린)
- 서브 액센트: #FF2E97 (네온 핑크)
- 카드 배경: #111118
- 제목 폰트: Black Han Sans
- 본문 폰트: Noto Sans KR
- 숫자/코드: JetBrains Mono
```

## 보내는 방법

### 파일이 작을 때 (각 500줄 이하)
프롬프트 + 두 코드를 한 번에 보내면 됩니다.

### 파일이 클 때
단계적으로 나눠서 요청하세요:

**1차 요청:** 카드 컴포넌트 개선 (투표 인라인 + 애니메이션)
```
참조 코드에서 CaseCard, VoteBar, VoteButtons 컴포넌트를 참고해서
기존 [카드 컴포넌트 파일명]을 개선해줘.
```

**2차 요청:** 댓글 시스템 개선
```
참조 코드에서 InlineComments, CommentItem 컴포넌트를 참고해서
기존 [댓글 컴포넌트 파일명]을 개선해줘.
```

**3차 요청:** 전체 레이아웃 + 탭 필터
```
참조 코드에서 NeonsCourt 메인 컴포넌트의 탭 필터와 레이아웃을 참고해서
기존 [페이지 파일명]을 개선해줘.
```
